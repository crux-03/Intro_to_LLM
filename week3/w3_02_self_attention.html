Week 3 · Lesson 2

Self-Attention

Basic attention uses embeddings directly as Q, K, V. Self-attention adds learnable projections — giving the model the freedom to decide what to look for, what to expose, and what to return.

Part 01

Why Learnable Projections?

In basic attention, Q = K = V = the embeddings themselves. This means each word's query is its embedding, and its key is its embedding. There's no flexibility — the model can't decide what aspects of a word to use for searching versus matching versus returning.

Self-attention introduces three learnable weight matrices that transform the embeddings:

# Basic attention (Lesson 1)
Q = embeddings          # query IS the embedding
K = embeddings          # key IS the embedding
V = embeddings          # value IS the embedding

# Self-attention (this lesson)
Q = embeddings @ W_q    # query is a LEARNED function of embedding
K = embeddings @ W_k    # key is a LEARNED function of embedding
V = embeddings @ W_v    # value is a LEARNED function of embedding

Now the model can learn what features to extract for each role. W_q might learn to extract "what grammatical role am I?" while W_k might learn to extract "what can I offer to others?"

🎭 Analogy — Job Interview: Basic attention is like everyone wearing the same name tag. Self-attention is like having three different cards: your question card (what you're looking for in a colleague), your badge (what you advertise about yourself), and your resume (your actual skills). Each card serves a different purpose, and you learn to fill them in strategically.

The key insight: W_q, W_k, and W_v are all learned during training. The model discovers on its own what features are useful for each role. After training on enough text, these matrices specialize beautifully.

🔢 Implementation detail: In PyTorch, each projection is just nn.Linear(d_model, d_model) — a matrix multiply plus an optional bias. So W_q, W_k, W_v are each (d_model × d_model) matrices with d_model bias terms.
Part 03

The Self-Attention Module

Here's the complete self-attention module — the core building block of every transformer:

class SelfAttention(nn.Module):
    def __init__(self, d_model, dropout=0.1):
        super().__init__()

        # Learnable projections
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)

        # Output projection
        self.W_o = nn.Linear(d_model, d_model)

        # Dropout for regularization
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        Q = self.W_q(x)          # (batch, seq, d_model)
        K = self.W_k(x)
        V = self.W_v(x)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(d_model)
        weights = F.softmax(scores, dim=-1)
        weights = self.dropout(weights)

        attended = weights @ V
        output = self.W_o(attended)   # output projection
        return output

Notice the four projection matrices: W_q, W_k, W_v for input projections, and W_o for the output projection. The input and output shapes are identical — (batch, seq_len, d_model) — which means self-attention layers can be stacked.

Part 04

The Output Projection (W_o)

After computing the weighted combination of values, we apply one more linear transformation: W_o. You might wonder: why do we need another projection at the end?

Mixing
Combines information from the attended values in new ways
Capacity
More parameters = more expressive model
Multi-Head
In multi-head attention (next lesson), W_o combines info from all heads
Symmetry
Input projections (W_q,W_k,W_v) → Attention → Output projection (W_o)

Think of it as a "final filter" — after gathering information from other positions, W_o decides how to combine and present that information.

Part 05

Parameter Count

How many parameters does self-attention have? Each projection is a (d_model × d_model) matrix plus a bias vector of size d_model. With 4 projections, that's:

# Parameters per projection: d_model × d_model + d_model (bias)
# Total: 4 × (d_model² + d_model)
# ≈ 4 × d_model²  (bias is negligible)

See how parameters scale with model dimension:

📊 Scaling: Parameters grow as O(d_model²). Doubling d_model quadruples the parameters. GPT-2 Small (d=768) has ~2.4M parameters just in one attention layer's projections. GPT-3 (d=12288) has ~600M per attention layer!

Part 06

Dropout in Attention

We apply dropout to the attention weights during training. This randomly zeros out some attention connections, forcing the model to not rely too heavily on any single connection.

▸ Dropout Visualization

Click to see how dropout randomly masks attention connections during training:

During evaluation (model.eval()), dropout is disabled — all connections are active. This means the same input always produces the same output in eval mode, but produces different outputs in training mode (due to random dropout).
