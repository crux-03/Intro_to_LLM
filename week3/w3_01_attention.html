Week 3 · Lesson 1

Understanding Attention

Attention is the breakthrough that made transformers possible — the mechanism that lets every word directly "look at" every other word and decide which ones matter.

Part 01

The Problem Attention Solves

Consider this sentence: "The cat sat on the mat because it was tired." What does "it" refer to? The cat, obviously. For us, this is trivial. For a model, it's surprisingly hard.

"it" is 7 words away from "cat", and there's another noun ("mat") in between. The model needs to somehow connect these distant words and figure out the correct reference.

▸ RNN vs Attention — Distance Problem

Click a word to see how each approach connects it to the rest of the sentence:

Old approach (RNNs): Process words one at a time, passing information forward through hidden states. Information gets "diluted" over distance — by the time you reach "it", the signal from "cat" is faint.

New approach (Attention): Let every word directly look at every other word. Compute "how relevant is each word to my current prediction?" Distance doesn't matter — "it" can attend directly to "cat" regardless of how far apart they are.

🔍 Analogy — Searching a Room: An RNN is like looking for your keys by checking each room in order, one at a time, and trying to remember what you saw. Attention is like having a bird's-eye view of every room simultaneously — you can instantly spot the keys no matter which room they're in.
Part 02

The Query-Key-Value Framework

Attention uses three concepts: Query (Q), Key (K), and Value (V). Think of it like searching a library:

Query (Q)
What you're looking for: "books about cats"
Key (K)
The label on each book: "animals", "cooking"
Value (V)
The actual content of each book
Score
How well each key matches your query

The process works in three steps:

1
Compare your Query to all Keys — measure similarity
2
Convert similarities to attention weights (probabilities that sum to 1)
3
Use weights to create a weighted combination of Values

Part 03

The Attention Formula

The complete attention mechanism fits in a single formula:

Attention(Q, K, V) = softmax(Q × KT / √d) × V

Let's break this down piece by piece:

1
Q × Kᵀ — Dot product between query and all keys. Measures similarity: higher = more similar.
2
/ √d — Scale by square root of dimension. Prevents scores from getting too large (which would make softmax "peaky").
3
softmax(...) — Convert scores to probabilities that sum to 1. These are the attention weights.
4
× V — Weight values by attention weights. Return weighted combination.

🔢 Why scale by √d? Without scaling, as the dimension d grows, dot products get larger (they sum more terms). Large values pushed through softmax produce near-one-hot distributions — almost all weight goes to a single position, making gradients tiny. Dividing by √d keeps the variance of the scores at ~1 regardless of dimension.

Part 05

Implementation in Code

The attention function is surprisingly compact — just a few lines of code:

def attention(Q, K, V):
    """Basic attention mechanism."""
    d_k = K.shape[-1]

    # Step 1-2: Compute scaled attention scores
    scores = Q @ K.T / math.sqrt(d_k)

    # Step 3: Convert to probabilities
    weights = F.softmax(scores, dim=-1)

    # Step 4: Weighted combination of values
    output = weights @ V

    return output, weights

For basic attention, Q = K = V = the input embeddings. The function returns both the output (weighted combination of values) and the attention weights (for visualization).

Handling Batches

In practice, we process multiple sequences at once. The shapes become (batch_size, seq_length, d_model) and we need to transpose the last two dimensions of K:

# Batched attention — same formula, extra batch dimension
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
# K.transpose(-2, -1): (batch, seq, d) → (batch, d, seq)
# Result: (batch, seq, seq) — attention matrix per sequence

Part 06

Why Attention Is So Powerful

Attention was the key insight in the landmark paper "Attention Is All You Need" (2017). It gives models four crucial abilities:

Long-Range
Every word directly attends to every other word. Word 1 can easily attend to word 100 — distance doesn't matter.
Parallel
All attention computations happen simultaneously. GPUs love this — much faster than sequential RNNs.
Interpretable
Attention weights show what the model "looks at." We can visualize which words relate to which.
Flexible
Same mechanism works for any sequence length. Can be adapted for many different tasks.
💡 Key insight: This is why the paper was called "Attention Is All You Need" — the authors showed that attention alone (without recurrence or convolution) was sufficient to achieve state-of-the-art results. Every modern LLM — GPT, Claude, Gemini — is built on this foundation.
