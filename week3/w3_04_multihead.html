Multi-Head Attention

Instead of one attention pattern, we run multiple "heads" in parallel. Each head can learn to focus on different relationships — syntax, semantics, position — simultaneously.

Part 01

Why Multiple Heads?

Consider: "The cat sat on the mat because it was tired."

Multiple relationships matter simultaneously: "sat" relates to "cat" (subject-verb), "it" refers to "cat" (pronoun resolution), "tired" relates to "sat" (cause-effect). A single attention head might capture one of these. Multiple heads can capture all of them at once.

👥 Analogy — Committee of Experts: Imagine hiring one expert to analyze a document — they'd focus on whatever they know best and miss other aspects. Now imagine a committee: a grammarian spots syntax, a semantics expert finds meaning, a logician traces reasoning. Each contributes a different perspective, and together they understand the document far better than any one alone. Multi-head attention is this committee.

Part 02

The Reshape Trick

We could create separate W_q, W_k, W_v for each head — but that's wasteful. Instead, we use one big projection then reshape into heads:

1
Project: (batch, seq, d_model) → (batch, seq, d_model)
2
Reshape: (batch, seq, d_model) → (batch, seq, num_heads, head_dim)
3
Transpose: → (batch, num_heads, seq, head_dim)

The key insight: head_dim = d_model / num_heads. We're not adding computation — we're splitting the same dimensions into parallel streams.

Part 04

The Combined QKV Projection

For efficiency, we combine all three projections into one matrix multiply:

# Instead of 3 separate matmuls:
Q = W_q(x)    # matmul 1
K = W_k(x)    # matmul 2
V = W_v(x)    # matmul 3

# One combined matmul + reshape:
qkv = W_qkv(x)            # single matmul!
Q, K, V = split(qkv, 3)   # just reshaping

Same result, fewer operations. GPUs prefer fewer, larger operations over many small ones.

Part 05

Complete Implementation

class MultiHeadCausalAttention(nn.Module):
    def __init__(self, d_model, num_heads, max_seq=512):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads
        # Combined QKV projection
        self.qkv_proj = nn.Linear(d_model, 3 * d_model)
        self.out_proj = nn.Linear(d_model, d_model)
        mask = torch.tril(torch.ones(max_seq, max_seq))
        self.register_buffer('mask', mask)

    def forward(self, x):
        B, T, C = x.shape
        # Project Q,K,V in one shot
        qkv = self.qkv_proj(x)
        qkv = qkv.view(B, T, 3, self.num_heads, self.head_dim)
        qkv = qkv.permute(2,0,3,1,4)  # (3,B,H,T,D)
        Q, K, V = qkv[0], qkv[1], qkv[2]

        # Per-head attention with causal mask
        scores = Q @ K.transpose(-2,-1) / math.sqrt(self.head_dim)
        scores = scores.masked_fill(self.mask[:T,:T]==0, float('-inf'))
        weights = F.softmax(scores, dim=-1)

        # Combine heads
        out = (weights @ V).transpose(1,2).reshape(B, T, C)
        return self.out_proj(out)
Key shapes: After splitting into heads, each head operates on (batch, num_heads, seq, head_dim). The attention matrix is (batch, num_heads, seq, seq) — one attention pattern per head. After combining, we're back to (batch, seq, d_model).
