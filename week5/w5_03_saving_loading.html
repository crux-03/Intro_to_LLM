Saving & Loading Models

State dictionaries, checkpoints, pretrained weights, and transfer learning — how to persist and reuse trained models.

Part 01

State Dict

Every PyTorch model has a state_dict — a dictionary mapping layer names to their parameter tensors. This is the model's "brain".

# View all parameters
for name, param in model.state_dict().items():
    print(name, param.shape)
# fc1.weight  (20, 10)
# fc1.bias    (20,)
# fc2.weight  (5, 20)
# fc2.bias    (5,)

Part 02

Saving & Loading

# SAVE (recommended: state_dict only)
torch.save(model.state_dict(), 'weights.pt')

# LOAD
model = GPT(...)  # Same architecture!
model.load_state_dict(torch.load('weights.pt'))
Never save the whole model with torch.save(model, ...). It uses pickle, which is fragile and a security risk. Always save just the state_dict.
🧩 What's the recommended way to save a PyTorch model?
A torch.save(model.state_dict(), path)
B torch.save(model, path)
C model.save(path)
Part 03

Training Checkpoints

Save more than just weights — include optimizer state, epoch, loss, and config so you can resume training exactly where you left off.

checkpoint = {
    'epoch': epoch,
    'model_state_dict': model.state_dict(),
    'optimizer_state_dict': optimizer.state_dict(),
    'loss': loss,
    'config': model_config,
}
torch.save(checkpoint, 'checkpoint.pt')
📚 Analogy: Saving just weights is like saving a bookmark. Saving a checkpoint is like saving your entire reading session: the page, your notes, your reading speed, everything needed to resume perfectly.
Part 04

Pretrained Models

Training GPT-3 cost ~$4.6M. Instead, use pretrained weights that others have released!

# Hugging Face makes it easy
from transformers import GPT2LMHeadModel, GPT2Tokenizer

model = GPT2LMHeadModel.from_pretrained('gpt2')
tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
▸ Popular Open Models
Transfer learning: Start from pretrained weights, then fine-tune on your specific task. Much faster and often better than training from scratch. This is how most LLM applications work in practice.
🧩 Why use pretrained models?
A They're always more accurate
B Training from scratch is expensive; pretrained models leverage billions of tokens of prior learning
C Pretrained models don't need fine-tuning
