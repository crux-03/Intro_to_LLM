Evaluating Language Models

Cross-entropy loss, perplexity, train/val/test splits, and detecting overfitting — how to know if your model is actually good.

Part 01

Cross-Entropy Loss

Loss measures how surprised the model is by the correct answer. If the model assigns high probability to the right token, loss is low. If it's surprised, loss is high.

loss = -log(P(correct_token))
# Confident & correct:  -log(0.9) = 0.10
# Surprised:            -log(0.1) = 2.30
▸ Loss vs Confidence
Hover over the curve to explore.
📚 Analogy: Imagine betting on a horse race. If you bet big on the winner, your "surprise" is low. If the winner was a long shot, you're very surprised. Cross-entropy measures exactly this kind of surprise.

Part 02

Perplexity

Perplexity converts loss into something intuitive: PPL = exp(loss)

It answers: "How many tokens is the model effectively choosing between?" A perplexity of 10 means the model is as confused as randomly picking from 10 equally likely options.

▸ Loss ↔ Perplexity Calculator
Benchmarks: Random (50K vocab) ≈ 50,000 PPL. N-gram ≈ 200–500. LSTM ≈ 50–100. GPT-2 ≈ 15–30. Lower is always better.

Part 05

Evaluation Code

@torch.no_grad()     # No gradients = faster
def evaluate(model, data_loader):
    model.eval()          # Disable dropout
    total_loss = 0
    total_tokens = 0
    for x, y in data_loader:
        logits, loss = model(x, y)
        total_loss += loss.item() * y.numel()
        total_tokens += y.numel()
    avg_loss = total_loss / total_tokens
    return avg_loss, math.exp(avg_loss)
Two key details: @torch.no_grad() disables gradient tracking (faster, less memory). model.eval() turns off dropout so evaluation is deterministic.
