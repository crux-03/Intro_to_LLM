Part 01

Tokenizer with Special Tokens

Our tokenizer needs special tokens to mark the boundaries between user input, assistant output, and sequence end.

class InstructionTokenizer:
    special_tokens = ['<PAD>', '<UNK>', '<USER>', '<ASST>', '<END>']

    def encode_instruction(self, instruction, response):
        # Format: <USER> instruction <ASST> response <END>
        tokens = ['<USER>'] + tokenize(instruction) + \
                 ['<ASST>'] + tokenize(response) + ['<END>']

    def encode_prompt(self, instruction):

        return token_ids

Part 02

Dataset with Loss Masking

The dataset creates input_ids and labels tensors. Labels have -100 for prompt positions (masked) and actual token IDs for response positions.

class InstructionDataset(Dataset):
    def __init__(self, instructions, tokenizer, max_length):
        for instruction, response in instructions:
            ids, resp_start = tokenizer.encode_instruction(
                instruction, response, max_length
            )
            # Create masked labels
            labels = [-100] * resp_start + ids[resp_start:]
            #         ^^^^^ masked prompt    ^^^^^ actual response
            self.data.append({
                'input_ids': torch.tensor(ids),
                'labels': torch.tensor(labels)
            })

Part 04

Generation: Answering Questions

After training, generation works by encoding the prompt, then autoregressively sampling tokens until <END> is produced.

def chat(model, tokenizer, instruction):
    # 1. Encode prompt
    prompt_ids = tokenizer.encode_prompt(instruction)
    input_ids = torch.tensor([prompt_ids])

    # 2. Generate token by token
    output = model.generate(
        input_ids,
        max_new_tokens=30,
        temperature=0.7,
        end_token_id=tokenizer.end_id  # Stop at <END>
    )

    # 3. Decode response (skip prompt tokens)
    response_ids = output[0, len(prompt_ids):].tolist()
    return tokenizer.decode(response_ids)

Summary

Key Takeaways

Special Tokens
<USER>, <ASST>, <END> mark boundaries between instruction, response, and sequence end.
Loss Masking
Labels use -100 for prompt positions. ignore_index=-100 in cross_entropy skips them.
Same Architecture
Same GPT model as pretraining. Only the loss computation changes.
Generation
Encode prompt, generate until <END>, decode response. Temperature controls randomness.
