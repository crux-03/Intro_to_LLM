Week 7 · Lesson 1

Instruction Fine-Tuning

The secret sauce that transforms a text predictor into ChatGPT — teaching a model to follow instructions and generate helpful responses.

Part 01

The Journey to ChatGPT

ChatGPT didn't appear from nowhere. It's the result of three training stages, each building on the previous one:

1
Pretraining (Weeks 4-5): Train on massive text data. The model learns language patterns, grammar, facts, and reasoning. Result: great at completing text, but doesn't follow instructions.
2
Supervised Fine-Tuning (SFT) (This week!): Train on instruction-response pairs. The model learns to follow commands. Result: follows instructions like ChatGPT.
3
RLHF (Advanced): Reinforcement Learning from Human Feedback. Humans rate responses and the model learns preferences. Result: more helpful, harmless, and honest.
▸ Base Model vs Instruction-Tuned
Analogy: A base model is like a brilliant librarian who has read every book but only knows how to continue sentences. Instruction tuning teaches that librarian to actually answer questions and help people. Same knowledge, different behavior.

Part 02

Training Data Format

Instruction tuning uses pairs of (instruction, response). We format them with special tokens so the model knows what's the user's input and what's the expected output.

# Format: <USER> instruction <ASST> response <END>

"<USER> What is the capital of France? <ASST> The capital of France is Paris. <END>"
"<USER> Say hello in Spanish. <ASST> Hola! <END>"
"<USER> What is 2 plus 2? <ASST> 2 plus 2 equals 4. <END>"

Part 03

Loss Masking: The Key Insight

The most important technical detail: we only compute loss on response tokens, not the prompt. This teaches the model to generate responses, not to predict what the user will say.

▸ Interactive Loss Masking Visualization
# In PyTorch, -100 means "ignore this position"

labels = [-100] * prompt_length + response_ids
loss = F.cross_entropy(logits, labels, ignore_index=-100)

# Loss is ONLY computed on response tokens!
# The model learns: "given <USER> ... <ASST>, generate the right response"
Without loss masking, the model would try to predict the user's instruction too, which wastes capacity and can hurt response quality. It would spend effort learning to predict questions rather than learning to generate answers.

Part 04

Why Instruction Tuning Works

The pretrained model already has the knowledge. It knows that Paris is the capital of France, that 2+2=4, and what a poem looks like. Instruction tuning just teaches it a new pattern: "When you see <USER> ... <ASST>, generate a helpful response."

▸ Knowledge vs Pattern

This is why instruction tuning needs relatively little data — thousands of examples, not billions. You're not teaching new facts, just a new format for accessing existing knowledge.

Summary

Key Takeaways

Three Steps
Pretraining → SFT (instruction tuning) → RLHF. We focus on step 2.
Data Format
<USER> instruction <ASST> response <END> with special tokens.
Loss Masking
Only compute loss on response tokens. Use -100 to mask the prompt.
Task Diversity
QA, math, translation, creative, conversation. More types = more capable.
