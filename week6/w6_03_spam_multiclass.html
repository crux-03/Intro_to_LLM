Part B

Multi-class Classification

Binary classification has 2 classes. Multi-class classification has 3 or more. The architecture is nearly identical — just change num_classes!

# Binary: 2 classes
classifier = nn.Linear(d_model, 2)   # Spam/Ham

# Multi-class: 4 classes
classifier = nn.Linear(d_model, 4)   # Sports/Tech/Politics/Entertainment

# The loss function handles both automatically:
loss = F.cross_entropy(logits, labels)  # Works for any num_classes!

Multi-class Evaluation: Per-class Metrics

With multi-class, we compute precision, recall, and F1 for each class separately, then average them.

▸ Per-class Metrics Example
Macro vs Micro averaging: Macro averages metrics across classes equally (treats each class as equally important). Micro averages across all samples (larger classes dominate). Use macro when class sizes are imbalanced and you care about minority class performance.

Part C

Binary vs Multi-class Comparison

▸ Side-by-Side Comparison
Binary
2 classes (positive/negative, spam/ham). Output: 2 logits, argmax for prediction.
Multi-class
3+ classes. Same architecture, just more output units. Per-class + macro metrics.
Softmax
Converts logits to probabilities that sum to 1. Works for any number of classes.
Cross-Entropy
The standard loss for both. Measures how far predicted distribution is from the true label.

Summary

Key Takeaways

Spam Detection
Binary classification with urgency/money signals. Precision often prioritized over recall.
Multi-class
Same architecture as binary, just more output units. Cross-entropy handles it automatically.
Per-class Metrics
Compute P/R/F1 for each class, then macro-average. Reveals per-class strengths and weaknesses.
Practical Value
Spam filters, content moderation, news categorization, ticket routing — these are real production tasks.
