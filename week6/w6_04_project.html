Early Stopping

The project uses early stopping to prevent overfitting. Training stops when validation accuracy hasn't improved for patience epochs, and the best model weights are restored.

def train(model, train_data, val_data, epochs=20, patience=5):
    best_acc = 0
    best_state = None
    no_improve = 0

    for epoch in range(epochs):
        # ... train ...
        acc = validate(model, val_data)

        if acc > best_acc:
            best_acc = acc
            best_state = model.state_dict()  # Save best
            no_improve = 0
        else:
            no_improve += 1

        if no_improve >= patience:
            print("Early stopping!")
            break

    model.load_state_dict(best_state)  # Restore best
Why early stopping matters: Without it, the model continues training past the optimal point, memorizing training data instead of learning generalizable patterns. With it, you automatically get the best model without manually tuning the number of epochs.
