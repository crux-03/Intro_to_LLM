Part 01

Pretraining vs Fine-Tuning

Pretraining taught the model general language understanding: grammar, syntax, word meanings, world knowledge, and reasoning patterns. Fine-tuning teaches the model your specific task: "Is this review positive?", "Is this email spam?", "What topic is this article about?"

The magic of fine-tuning is that it's fast and needs little data. Why? Because the model already understands language. It just needs to learn to apply that understanding to your task.

Analogy — Education: Pretraining is like K-12 education: years of learning reading, writing, math, and general knowledge. Fine-tuning is like specialized training: a med student already knows biology, they just need to learn clinical practice. You don't start from scratch — you build on existing knowledge.

Part 02

Transfer Learning Illustrated

What happens inside the model during fine-tuning? The key insight is that lower layers learn general features (grammar, syntax patterns) while higher layers learn task-specific features. Fine-tuning mostly adjusts the higher layers.

This is why transfer learning works so well. The lower layers that learned basic language patterns transfer directly to the new task. Only the higher layers need significant adjustment.

Real-world impact: GPT-2 was pretrained for months on millions of documents costing millions of dollars. Fine-tuning takes hours on thousands of examples and costs almost nothing. The pretrained knowledge transfers to the new task!
Part 03

Adding a Classification Head

For classification, we add a new linear layer on top of GPT. This is the "classification head" — it converts GPT's hidden states into class predictions.

1
GPT processes input left-to-right, producing hidden states for each token: (batch, seq_length, d_model)
2
Extract the last token's hidden state. It has "seen" all previous tokens, so it contains a summary of the whole sequence: (batch, d_model)
3
Classification head maps from d_model to num_classes: Linear(d_model, num_classes) → (batch, num_classes)
class GPTClassifier(nn.Module):
    def __init__(self, gpt, num_classes):
        self.gpt = gpt
        d_model = gpt.token_embedding.weight.shape[1]
        self.classifier = nn.Linear(d_model, num_classes)

    def forward(self, x):
        hidden = self.get_hidden_states(x)  # (batch, seq, d_model)
        last = hidden[:, -1, :]              # (batch, d_model)
        return self.classifier(last)          # (batch, num_classes)

Part 04

Freezing Strategies

How much of the model should we train? There are three main strategies, each with different tradeoffs between speed, data requirements, and accuracy.

▸ Interactive Freezing Comparison

Click each strategy to see what's trainable vs frozen:

Head Only
~0.04% trainable. Fastest training, least data needed. Good for quick prototyping.
Top Layers
~25% trainable. Balance of speed and accuracy. Good with moderate data.
Full Fine-Tuning
100% trainable. Best accuracy but slowest. Needs lots of data to avoid overfitting.
Gradual Unfreezing
Start with head only, then progressively unfreeze more layers. Prevents catastrophic forgetting.
# Freeze all GPT parameters
for param in model.gpt.parameters():
    param.requires_grad = False

# Only the classifier head is trainable now!
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total = sum(p.numel() for p in model.parameters())
print(f"Trainable: {trainable}/{total}")  # ~130 / ~365,000

Summary

Key Takeaways

Transfer Learning
Pretrained model already understands language. Fine-tuning just teaches the new task.
Classification Head
A simple Linear layer on top of GPT. Uses the last token's hidden state.
Freezing Strategies
Head only (~0.04%), top layers (~25%), or full (100%). Trade speed vs accuracy.
Why It Works
Lower layers keep general features. Higher layers adapt. New head learns the task.
