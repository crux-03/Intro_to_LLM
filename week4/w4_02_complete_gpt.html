Part 02

Weight Tying

A key trick: the input embedding and output projection share the same weight matrix.

self.lm_head.weight = self.token_embedding.weight  # Same matrix!
📚 Analogy — Dictionary: The embedding matrix is like a dictionary: look up a word → get its meaning (vector). The output projection is reading the dictionary in reverse: given a meaning (vector) → find the closest word. Using the same dictionary for both directions makes sense and saves parameters!

Part 03

The Forward Pass

▸ Trace Data Through GPT
def forward(self, input_ids, targets=None):
    B, T = input_ids.shape
    tok_emb = self.token_embedding(input_ids)  # (B,T,d)
    pos_emb = self.position_embedding(
        torch.arange(T, device=input_ids.device))  # (T,d)
    x = self.dropout(tok_emb + pos_emb)
    for block in self.blocks:
        x = block(x)
    x = self.ln_final(x)
    logits = self.lm_head(x)  # (B,T,vocab_size)
    loss = None
    if targets is not None:
        loss = F.cross_entropy(
            logits.view(-1, logits.size(-1)),
            targets.view(-1))
    return logits, loss

Part 05

Text Generation

The generation loop is beautifully simple:

1
Get logits for the last position
2
Divide by temperature (controls randomness)
3
Apply softmax to get probabilities
4
Sample next token from distribution
5
Append and repeat!
for _ in range(max_new_tokens):
    logits, _ = model(input_ids[:, -max_seq:])
    logits = logits[:, -1, :] / temperature
    probs = F.softmax(logits, dim=-1)
    next_token = torch.multinomial(probs, num_samples=1)
    input_ids = torch.cat([input_ids, next_token], dim=1)
